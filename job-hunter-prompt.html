Step-by-Step Build Guide
1
Database — PostgreSQL Schema
Set up your PostgreSQL database with tables for jobs and your scrape logs. The jobs table needs columns for everything you'll filter on later — title, company, URL, source, score, region, salary, and a status field for your pipeline.
prompt-database.txt
Write me a PostgreSQL schema for a job aggregator system. I need these tables: 1. jobs table with columns: - id (serial primary key) - title, company, url (unique constraint on url for dedup) - source (which site it came from: linkedin, indeed, remoteok, etc.) - score (integer, calculated by scraper) - region (e.g. "remote", "india", "europe", "us") - salary_min, salary_max (nullable integers) - status (enum: new, bookmarked, applied, interview, offer) - scraped_at, created_at timestamps - description (text, nullable) - tags (text array for skills matched) 2. scrape_logs table to track each scrape run: - id, source, jobs_found, jobs_new, started_at, finished_at, error (nullable) Include indexes on: url, status, score, scraped_at, source. Default status should be 'new'. Add a comment explaining the dedup strategy.
Your homework: Figure out the right index strategy for your query patterns. Think about whether you want a composite index on (status, score) for the dashboard's default view. Research PostgreSQL partial indexes for the cleanup query.
2
Config — Your Skills & Preferences
The scraper reads a config file to know what to search for and how to score. Define your primary skills, secondary skills, target industries, and region preferences. This is what makes the system personal.
prompt-config.txt
Create a config.json structure for my job scraper. I need: { "keywords": { "primary": ["react", "angular", "vue", "typescript", "nextjs"], "secondary": ["node", "python", "go", "graphql", "tailwind"], "industries": ["AI", "fintech", "healthtech", "edtech", "saas"] }, "scoring": { "primary_skill_match": 3, "secondary_skill_match": 1, "industry_match": 2, "region_boost": 3, "preferred_regions": ["remote", "india", "europe"] }, "sources": { "apify": { "linkedin": { "actor_id": "..." }, "indeed": { "actor_id": "..." }, "naukri": { "actor_id": "..." } }, "free_apis": ["remoteok", "arbeitnow", "remotive", "jobicy", "hackernews", "adzuna"] }, "schedule": { "scrape_time": "18:00", "morning_brief": "08:00", "evening_digest": "22:00" }, "cleanup": { "purge_after_days": 7, "keep_statuses": ["bookmarked", "applied", "interview", "offer"] } } Write a Python function that loads this config and validates all required fields exist. Include type hints.
Your homework: Research the right Apify actors for LinkedIn/Indeed/Naukri job scraping. Actor IDs change — search the Apify store for the latest ones. Customize keywords for your actual skill set.
3
Python Scraper — The Data Engine
This is the core. A Python script that runs as a cron job at 6PM daily. It hits all 10 sources, normalizes the data, scores each job based on your config, deduplicates by URL, and inserts into PostgreSQL.
prompt-scraper.txt
Build a Python job scraper (scraper.py) that: 1. SOURCES — Fetches jobs from these sources: a. Apify actors (for LinkedIn, Indeed, Naukri) — use the Apify Python client b. Free APIs: - RemoteOK: GET https://remoteok.com/api (returns JSON array) - Arbeitnow: GET https://www.arbeitnow.com/api/job-board-api - Remotive: GET https://remotive.com/api/remote-jobs - Jobicy: GET https://jobicy.com/api/v2/remote-jobs - HackerNews (Who is Hiring): research the HN API for monthly threads - Adzuna: GET https://api.adzuna.com/v1/api/jobs/{country}/search/1?app_id=X&app_key=Y 2. NORMALIZE — Each source returns different JSON shapes. Map every source to a common structure: { title, company, url, source, region, salary_min, salary_max, description } 3. SCORE — For each job, read config.json and: - Check title + description against primary keywords (+3 each match) - Check against secondary keywords (+1 each) - Check company/description against industries (+2) - If region matches preferred_regions (+3) - Cap at a reasonable max 4. DEDUP — Before insert, check if URL already exists in the database. Skip if it does. 5. INSERT — Batch insert new jobs into PostgreSQL. Log the scrape run to scrape_logs. 6. Use asyncio + aiohttp for parallel fetching across sources. 7. Handle errors gracefully — if one source fails, continue with others. Log errors. 8. Load .env for DATABASE_URL, APIFY_TOKEN, ADZUNA_APP_ID, ADZUNA_APP_KEY. Include requirements.txt with dependencies. Include the crontab line to run this at 6PM daily.
Your homework: The free APIs listed above may change their endpoints or rate limits. Test each one manually with curl before integrating. The HackerNews "Who is Hiring" scraping needs you to figure out how to find the monthly thread ID — that's an interesting parsing challenge. Apify actors have their own input schemas — read each actor's docs.
4
Go Server — API + Slack Bot + Cleanup
A single Go binary that does three things: serves a REST API for the dashboard, runs a Slack chatbot via Socket Mode, and auto-cleans old jobs. Runs as a systemd service so it's always up.
prompt-go-server.txt
Build a Go server (main.go) that does 3 things in one binary: 1. REST API (net/http, no framework): - GET /api/jobs — list jobs with query params: status, region, search, sort (score/date), page, limit - PATCH /api/jobs/:id — update job status (new/bookmarked/applied/interview/offer) - GET /api/stats — counts by status, total jobs, today's new jobs - Serve the React build folder as static files at the root / - CORS headers for local development 2. SLACK BOT (Socket Mode): - Use the slack-go/slack library - Parse natural language commands: "today" → show today's top 5 new jobs "search react" → search jobs matching "react" "top" or "best" → highest scored jobs "stats" → show pipeline counts "scrape" → trigger a scrape run (exec the Python script) - Paginate results: 5 jobs per message, "more" for next page - Format with Slack Block Kit for nice rendering 3. AUTO-CLEANUP (goroutine): - Every hour, delete jobs where status='new' AND created_at < now() - 7 days - Never delete bookmarked/applied/interview/offer 4. NOTIFICATIONS (goroutine): - At 8AM: POST to Slack webhook with top 5 new jobs from last scrape - At 10PM: POST digest (summary counts + top jobs) via Slack + optional email Connect to PostgreSQL using pgx. Load config from .env file. Include a systemd unit file to run this as a service. Include go.mod with dependencies.
Your homework: The Slack Bot requires Socket Mode — research how to set up a Slack App with the right OAuth scopes (chat:write, commands, app_mentions:read, etc.). The NLP parsing is intentionally simple here — you can make it smarter. Figure out systemd unit file placement and how to auto-restart on crash.
5
React Dashboard — Your Command Center
A simple React app that shows all your jobs in a sortable, filterable table. Status dropdowns to move jobs through your pipeline. Served as static files by the Go server.
prompt-dashboard.txt
Build a React dashboard (Vite + React) for a job aggregator. It connects to a Go API at the same origin. FEATURES: - Job table with columns: Score, Title, Company, Region, Salary, Source, Status, Date - Status dropdown in each row: new / bookmarked / applied / interview / offer - Changing status sends PATCH /api/jobs/:id - Color-coded: new=gray, bookmarked=blue, applied=yellow, interview=purple, offer=green - Filters bar at top: - Search input (searches title + company) - Region dropdown (all, remote, india, europe, us, etc.) - Status filter - Sort by: score (desc), date (newest), salary (highest) - Stats bar showing: Total jobs, New today, Bookmarked, Applied, Interviews, Offers - Pagination: 25 jobs per page - Click on job title opens the URL in new tab - Dark theme (background #0a0a0a, cards #111, accent #4d8bff) - Responsive — works on mobile TECH: - Vite + React (no TypeScript needed, keep it simple) - fetch() for API calls, no axios - CSS modules or plain CSS, no Tailwind - Build output goes to ./build folder which Go serves as static Include package.json and vite.config.js. The build command should output to ../static or ./build.
Your homework: The dashboard design is up to you. You could add charts for application stats over time, a calendar view for interviews, or keyboard shortcuts for power users. Think about what UX would make YOUR job search faster.
6
Infrastructure — Nginx, mDNS, Health Checks
The last mile: make the dashboard accessible from any device on your LAN via a friendly URL, and set up health monitoring so crashed services auto-restart.
prompt-infra.txt
Set up the infrastructure for my job hunter system running on a Linux machine (Ubuntu/Debian): 1. NGINX CONFIG: - Reverse proxy on port 80 → Go server on :3000 - Include proper proxy headers (X-Real-IP, X-Forwarded-For) - Gzip compression for static assets - Server name: job.local 2. AVAHI / mDNS: - Install avahi-daemon so the machine broadcasts "job.local" on the LAN - Any device on the same network can open http://job.local - Include the avahi service file 3. HEALTH CHECK SCRIPT (bash): - Runs every 5 minutes via cron - Checks if the Go server is responding (curl localhost:3000/api/stats) - Checks if PostgreSQL is accepting connections - If anything is down, restart the systemd service - Send a Slack notification if a restart happens - Log all checks to /var/log/job-hunter-health.log 4. CRONTAB SUMMARY: - 6PM daily: run scraper.py - Every 5 min: run health check - Include the exact crontab lines Give me all config files I need to set this up.
Your homework: mDNS (Avahi) only works on local networks. Research how .local domains resolve on different OSes — macOS works natively, Windows needs Bonjour, Linux needs nss-mdns. If you want remote access, look into Tailscale or Cloudflare Tunnel. The health check script is basic — you could upgrade it to check scraper success rates, DB disk usage, etc.
🛠 Putting It All Together

🗒 Project Structure

  • scraper/ — Python scraper, config.json, requirements.txt
  • server/ — Go server, main.go, go.mod
  • dashboard/ — React app, src/, vite.config.js
  • infra/ — nginx.conf, systemd unit, health-check.sh, avahi service
  • .env — all your secrets (never commit this)

⚡ Quick Start Order

  • First: Set up PostgreSQL + create the schema
  • Second: Build the scraper, test with 1-2 free APIs first
  • Third: Build the Go server with just the REST API
  • Fourth: Build the React dashboard, connect to API
  • Fifth: Add the Slack bot to Go server
  • Sixth: Set up Nginx, Avahi, cron, health checks
  • Last: Add notifications (morning brief, evening digest)
