The Complete Guide 📖

Follow each step. By the end, you'll have a fully functional, payment-ready e-commerce website.

Step 1 of 6

✍️ Sketch Your Wireframe on Paper

Grab a pen and paper. Draw out every screen your website needs. Don't worry about making it pretty — rough is perfect. The AI will handle the design.

  • Draw the Home page — hero banner, product grid, navigation, footer
  • Draw the Product page — image gallery, price, size selector, add to cart
  • Draw the Cart page — items list, quantities, total, checkout button
  • Draw the Checkout page — address form, payment section, order summary
  • Draw the Collections/Category page — filters, sorted products
  • Optionally sketch About, Contact, Order tracking pages

Take clear photos of each sketch. Good lighting, straight angle. These photos become your AI input.

Pen + Paper FREE
Phone Camera FREE
Step 2 of 6

🤖 Convert Sketches to Prototype with AI

Upload your paper sketches to an AI tool that converts wireframes into working code. You'll get a clickable prototype with real HTML/CSS.

  • Go to Google Stitch (stitch.withgoogle.com) — upload your sketch photos
  • Or use Vercel v0 (v0.dev) — describe + upload your wireframe
  • Or use Bolt.new, Lovable, or Replit — all accept image inputs
  • Review the output — it won't be perfect, but it gives you a solid starting structure
  • Export/copy the generated code — you'll refine this in the next step

The prototype is your foundation. It captures your layout intent. Now we make it production-grade.

Step 3 of 6

🎨 Build the Production Frontend

Take the prototype code and feed it to Claude or ChatGPT. This is where the magic happens — AI turns your rough prototype into a polished, responsive, production-ready frontend.

  • Paste the prototype code into Claude or ChatGPT
  • Ask it to: "Polish this into a production e-commerce frontend with Tailwind CSS, smooth animations, mobile responsive, professional design"
  • Specify your brand colors, fonts, style — "luxury minimal like Dior" or "warm earthy like ethnic wear"
  • Add features: product filtering, size selection, image zoom, cart with localStorage, search
  • Make sure it's mobile-first — 80%+ of your traffic will be from phones (especially from Instagram)
  • Test on phone, tablet, desktop. Fix any responsive issues with AI
  • Pro tip: Give AI a reference website. "Make it look like this: [URL]" works surprisingly well
  • Pro tip: Use Claude Code (CLI) for iterating directly on your files — fastest workflow

    Step 4 of 6

    ⚙️ Set Up the Backend

    Your frontend needs a backend for product data, orders, user auth, and admin management. Here's how to structure it.

    • Choose your stack: Cloudflare Workers + D1 (free, serverless) or Node.js + PostgreSQL
    • Create API endpoints: /api/products, /api/cart, /api/orders, /api/auth
    • Set up OTP-based authentication — no passwords, just phone/email OTP (use MSG91 or similar)
    • Build an Admin panel — add/edit products, view orders, manage inventory
    • Set up email notifications — order confirmation, shipping updates (use Resend or Mailgun, both free tier)
    • Add order tracking — let customers check their order status
  • Use MVC architecture — separate routes, controllers, and models for clean code
  • AI can build your entire backend too. Give it your API spec and database schema

    Step 5 of 6

    💳 Integrate Razorpay Payments

    Razorpay is the easiest way to accept payments in India. UPI, cards, net banking, wallets — everything works. Here's the integration flow.

    • Create a Razorpay account at razorpay.com — KYC takes 1-2 days
    • Get your API Key ID and Key Secret from the Razorpay Dashboard
    • Backend: Create an order using Razorpay Orders API — POST /v1/orders with amount and currency
    • Frontend: Load Razorpay Checkout.js — open the payment modal with the order ID
    • Backend: Verify payment signature — compare razorpay_signature with HMAC SHA256 of order_id + payment_id
    • On success: Update order status in your database, send confirmation email, show success page
    • Set up Razorpay webhooks for payment.captured, payment.failed events as backup
  • Start with Razorpay Test Mode — use test API keys to simulate payments without real money
  • Always verify payments on the backend. Never trust frontend-only payment confirmation
  • Razorpay charges ~2% per transaction. No monthly fees. No setup cost

    🚀 Deploy & Go Live

    Everything's built. Time to deploy and share with the world.

    • Deploy frontend on Cloudflare Pages — npx wrangler pages deploy .
    • Deploy backend on Cloudflare Workers — npx wrangler deploy

    • Set up Google Analytics to track visitors and conversions
