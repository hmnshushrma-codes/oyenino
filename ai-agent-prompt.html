🧠 Prompt 1 — The AI Agent Brain
agent-prompt.txt
You are [YOUR_AGENT_NAME] — an AI-powered [YOUR_AGENT_ROLE] that helps [YOUR_TARGET_USER] make better decisions. ===== WHAT THIS AGENT DOES ===== [Describe what your agent does in 2-3 lines. Example: "This agent helps Indian farmers find the best mandi to sell their crops by comparing prices across nearby markets and calculating the actual amount they'll receive after all deductions."] ===== USER INPUT ===== The user will provide: - [INPUT_1]: [Description — e.g., "Product/item type"] - [INPUT_2]: [Description — e.g., "Quantity or budget"] - [INPUT_3]: [Description — e.g., "Location or preferences"] ===== YOUR JOB ===== 1. [TASK_1 — e.g., "Find 4-5 nearby options/providers/sources"] 2. For each option, calculate/estimate: - [METRIC_1]: [e.g., "Base price or cost"] - [METRIC_2]: [e.g., "Distance or delivery time"] - [METRIC_3]: [e.g., "Additional fees or charges"] - [METRIC_4]: [e.g., "Hidden costs or deductions"] 3. [CALCULATION — e.g., "net_value = base_value - fees - charges - costs"] 4. Rank all options by [RANKING_METRIC] (best first) 5. Give a short recommendation explaining WHY the top option is best ===== IMPORTANT RULES ===== - Use realistic data for [YOUR_DOMAIN — e.g., "Indian market prices"] - Include at least one nearby/cheap option and one premium/distant option - The "best" option after all calculations may NOT be the most obvious one - If language is "hi", respond recommendation in Hindi. Otherwise English. ===== RESPONSE FORMAT ===== Respond ONLY in this JSON format, no markdown, no extra text: { "results": [ { "name": "[Option name]", "[METRIC_1_KEY]": [value], "[METRIC_2_KEY]": [value], "[METRIC_3_KEY]": [value], "[METRIC_4_KEY]": [value], "net_value": [calculated value] } ], "recommendation": "[Your recommendation text]" } Return 4-5 results sorted by net_value (best first). All calculations must be mathematically correct.

📝 Placeholders kaise fill kare?

  • [YOUR_AGENT_NAME] — Agent ka naam, e.g. "FasalDeal", "PriceHunt", "DocFinder"
  • [YOUR_AGENT_ROLE] — Kya karta hai, e.g. "agricultural market advisor", "price comparison tool"
  • [YOUR_TARGET_USER] — Kiske liye hai, e.g. "Indian farmers", "online shoppers", "students"
  • [INPUT_1/2/3] — User kya dega, e.g. "crop type", "budget", "city"
  • [METRIC_1/2/3/4] — Kya compare karoge, e.g. "price", "distance", "fees", "delivery time"
  • [CALCULATION] — Final formula, e.g. "net = price - shipping - tax"
⚡ Prompt 2 — The Backend (give this to Claude/ChatGPT)
backend-prompt.txt
Build me a Cloudflare Worker that acts as a backend API for an AI-powered [YOUR_AGENT_DESCRIPTION] called "[YOUR_AGENT_NAME]". TECH STACK: - Cloudflare Worker (JavaScript) - Google Gemini API (model: gemini-2.5-flash) - No database needed for MVP THE WORKER SHOULD HAVE: 1. POST /api/analyze endpoint that accepts: { "[INPUT_1_KEY]": "[value]", "[INPUT_2_KEY]": [value], "[INPUT_3_KEY]": "[value]", "lang": "en" } 2. A detailed prompt for Gemini that: - Acts as [YOUR_AGENT_ROLE] - Finds [WHAT_TO_FIND — e.g., "4-5 nearby options"] - Calculates [WHAT_TO_CALCULATE — e.g., "net price after all deductions"] - Ranks results by [RANKING_METRIC] - Gives recommendation in requested language (en/hi) - Responds in JSON only 3. Call Gemini API at: https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent API key from env variable GEMINI_API_KEY 4. Parse response, RECALCULATE the math server-side (LLMs sometimes get arithmetic wrong), re-sort, return clean JSON 5. GET /api/health endpoint 6. CORS headers for all origins 7. Proper error handling — if Gemini fails, return 502 PROJECT STRUCTURE: [your-agent]-worker/ ├── wrangler.toml (with route for [YOUR_DOMAIN]/*) ├── package.json └── src/ ├── index.js (router + CORS) └── handlers/ ├── analyze.js (Gemini prompt + API call + response parsing) └── health.js DEPLOYMENT: wrangler deploy wrangler secret put GEMINI_API_KEY Give me complete, deployable code for all files.
🎨 Prompt 3 — The Frontend (give this to Claude/ChatGPT)
frontend-prompt.txt
Build me a React component (Vite project) for an AI-powered [YOUR_AGENT_DESCRIPTION] called "[YOUR_AGENT_NAME]". DESIGN: - [THEME — e.g., "Light, warm, friendly" or "Dark, professional, sleek"] - Fully responsive — mobile-first (80%+ users on phone) - Clean, modern UI - Language toggle: English ↔ Hindi (bilingual UI) USER INPUT FORM: - [INPUT_1]: [Type — e.g., "Selection grid with icons" or "Dropdown" or "Text input"] - [INPUT_2]: [Type — e.g., "Number input"] - [INPUT_3]: [Type — e.g., "Text input with autocomplete"] - Submit button LOADING STATE: Step-by-step agent status animation while API responds: - [STEP_1_TEXT — e.g., "📡 Searching nearby options..."] - [STEP_2_TEXT — e.g., "✅ Found 4 options! Comparing..."] - [STEP_3_TEXT — e.g., "🧮 Calculating costs and fees..."] - [STEP_4_TEXT — e.g., "🏆 Best options ready!"] Each step appears one by one with a progress bar. Run API call and animation in parallel. RESULTS DISPLAY: - Result cards showing for each option: - Name + key identifier (e.g., distance, rating) - All metrics itemized (positive values in green, deductions in red) - NET VALUE highlighted prominently - Total value for user's quantity/budget - Best option highlighted with badge - Savings banner: "Extra [value] vs worst option" - AI recommendation with typewriter text effect FALLBACK: If API fails, show simulated demo data (hardcode 4 realistic results) API: - Read URL from import.meta.env.VITE_API_URL - Fallback to [YOUR_DEPLOYED_URL] OUTPUT: Complete Vite project: - package.json (with gh-pages for deploy) - vite.config.js (base: '/[YOUR_REPO_NAME]/') - index.html - src/main.jsx - src/[YourAgent].jsx - .env (VITE_API_URL=[YOUR_URL]) - .gitignore

🚀 Deploy kaise kare? 4 commands.

  • Backend: npx wrangler deploy → npx wrangler secret put GEMINI_API_KEY
  • Frontend: npm install → npm run deploy
  • Test: Open your GitHub Pages URL. Done. Live.
